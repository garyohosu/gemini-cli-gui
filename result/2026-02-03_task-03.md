## YOLO (-y) Expectation
- Current implementation in `server/gemini_server.js` forces `-y` (YOLO mode) on the `gemini` CLI subprocess.
- **Risk Identified**: If `gemini-cli` executes file operations directly due to `-y`, and the GUI then attempts to execute them again via `OperationExecutor` (after user approval), it may cause "File Already Exists" errors or redundant operations.
- **Recommendation**: Clarify if `gemini-cli` should be run without `-y` to allow the GUI to have exclusive control over execution, OR if `gemini-cli` has a mode to "output operations but do not execute".
